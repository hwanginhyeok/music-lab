# 야간 작업 — music-lab (2026-03-31)

## 배경
- Suno Pro 결제됨. WAV 다운로드 가능. 크레딧 뽕 뽑아야 함.
- Google OAuth 토큰 존재: client_secrets.json + token.json (YouTube upload + Drive readonly)
- Drive upload 스코프 추가 필요 (현재 readonly만)
- 목표: Suno 곡 대량 생성 → WAV 다운로드 → YouTube 업로드 파이프라인

## 1. Google Drive 스코프 확장
- token.json의 scopes에 drive.file 추가 필요 (현재 drive.readonly만)
- scripts/youtube_upload.py의 SCOPES 확인 후 drive.file 추가
- 재인증 필요할 수 있음 (token.json 삭제 → 재로그인)
- .env에 GOOGLE_CREDENTIALS_PATH=client_secrets.json 설정
- 주의: .env 수정은 GOOGLE_CREDENTIALS_PATH만 (SUNO_COOKIE 건드리지 않기)

## 2. Suno 자동화: Create 클릭 후 실제 생성 안 되는 문제 해결
- suno_client.py의 Create 버튼 클릭이 React form submit을 트리거하지 않는 문제
- 접근법 1: CDP(Chrome DevTools Protocol)로 Create 시 어떤 API 요청이 가는지 캡처
- 접근법 2: Suno의 실제 생성 API 엔드포인트를 찾아서 직접 호출 (쿠키 인증으로)
- 접근법 3: button.click() 대신 form submit 이벤트 직접 트리거
- 테스트: python3 suno_pipeline.py --title "Test" --style "bossa nova jazz" --lyrics "[Verse] test" --skip-drive
- 성공 기준: data/suno/ 에 파일 다운로드됨 + suno.com에서 크레딧 차감 확인

## 3. Suno 로그인 → 다운로드 → Drive/YouTube 업로드 파이프라인
- suno_download.py 신규 생성
- undetected-chromedriver로 suno.com 로그인 (쿠키 주입)
- suno.com/me 또는 라이브러리 페이지에서 곡 목록 가져오기
- WAV 다운로드 (Pro 계정이므로 WAV 가능)
- drive_uploader.py로 Google Drive 업로드
- scripts/youtube_upload.py 연계 — WAV → YouTube 업로드
- CLI: python3 suno_download.py --all (전체) 또는 --song-id UUID (개별)
- 파일명 규칙: {title}_{song_id}.wav
- db.py suno_songs 테이블에 메타데이터 저장

## 4. YouTube 업로드 자동화 연계
- scripts/youtube_upload.py 이미 존재
